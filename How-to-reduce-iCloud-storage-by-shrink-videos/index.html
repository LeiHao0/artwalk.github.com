How to reduce iCloud storage by shrinking videos

Posted on 2020-03-30 Edited on 2020-12-21 Disqus: Symbols count in article: 2.7k Reading time ≈ 2 mins.

In fact, subtracting unnecessary clips from 4k video can save space more effectively

When I gallery videos in “Photos”, it will show when and where I took them

This is a great feature that can help me recall my memories
But the original 4k takes up too much space in iCloud

For example, even with HEVC (High Efficiency Video Coding), a 2:30 video will occupy 1GB of space
My 200G iCloud plan is almost full ​​

Therefore, I plan to compress the video a bit to save some space.

but,

FFmpeg converts video, even if there is metadata, GPS will be lost

Fortunately, I found a way

Tools

  • FFmpeg
  • Exiftool

Export original video to folder

The common way to drag a video from “Photo” to a folder is to let “Photo” convert the video

Therefore, you must select all the videos you want to export,

File -> Export -> Export Unmodified Original For...

Shrink each video

iPhone default stream is

# 0: 0 (and): Video: hevc (Main) (hvc1 / 0x31637668), yuv420p (tv, bt709)

So using -pix_fmt yuv420p10le to re-encode to 10bit does not help

Then, you can copy and paste these codes into Terminal and cd VideoFolder, then shrinkVideos

Shrinking will take a long time

On my MacBook Pro (15 inches, 2018), it’s only 0.3 times
But usually, the video will be reduced by 5 times without losing too much quality

When all is done, the reduced video will be located under the shrunk/ folder

You can then compare them and delete the original video before importing the reduced video

Okay, in the end I can’t stand the extremely slow speed, so I use Nvidia 1080 GPU for acceleration to learn more: [CUDA GPU Accelerated h264/h265/HEVC Video Encoding with Staxrip](https://leihao0.github.io /CUDA-GPU-Accelerated-h264-h265-HEVC-Video-Encoding-with-Staxrip/)

The next step is to mark the converted video as hvc1, otherwise, an error will occur when importing photos:
