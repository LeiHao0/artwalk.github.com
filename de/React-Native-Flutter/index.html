Als iOS-Entwickler, der die Schritte  gerne befolgt, habe ich als iOS-Entwickler Swift, Storyboard und AutoLayout verwendet.

Ich denke, aus diesem Grund sind die meisten nativen Entwickler nicht einmal bereit, React Native zu lernen.

Vor zwei Jahren habe ich Mattermost gegründet und einige Änderungen in der vorherigen Firma vorgenommen, um Slack zu ersetzen. Dies war das erste React Native-Projekt, mit dem ich in Kontakt kam.

Leider ist es sehr leicht, Code zu verlieren, und jedes Mal, wenn ich etwas ändern möchte, muss ich das gesamte Projekt durchsuchen. Es gibt kein Storyboard, das mich darüber informiert, wer wem erzählt hat.

Ich unterhielt mich mit meinem ehemaligen Kollegen, der ein fortgeschrittenes Frontend war. Er half mir beim Starten des Update-Servers, der auch in Node.js geschrieben wurde. Ich habe ihm dieses Projekt übergeben, als ich ging.

Ihm zufolge liegt dies daran, dass er nicht weiß, wie man nativen Code schreibt, genau wie React React React.js ähnelt. Warum sollte man es nicht verwenden, um sowohl iOS als auch Android zu testen? Dann fragte ich ihn, ob er wisse, wie man sich schnell entwickelt und welche Art von Technologie er möchte. Sie können die Antwort erraten 😄.

Aus meiner Sicht ist es manchmal schwierig, Funktionen auch in schnellem Code zu implementieren. Dies ist definitiv ein langer Weg, um eine andere Technologie zu erreichen.

In diesen Tagen habe ich dann die Möglichkeit, eine Demo mit verschachtelten Magnet-Scroll-Ansichten, Schaltflächen und Navigation durchzuführen. Anstatt nativen Code zu verwenden, verließ ich meine Komfortzone und versuchte es mit React Native und Flutter.

  • Ansichten
    • ListView, ScrollView, Labels, Texte, TextFields
  • Aktionen
    • Ziehen, klicken, scrollen
    • Callback Block oder Delegate on View-Ereignisse

Wenn Sie sich mit einem von ihnen auskennen, bleibt nur die Art und Weise, wie Google es in das Framework implementiert oder das Handbuch liest.

Native reagieren

Das Beeindruckende an React Native ist ** state **

Dies kann durch das didSet der Variablen in Swift erreicht werden, und es gibt viele Frameworks, die dies tun können, wie z. B. RxSwift, PromiseKit

Flattern

Früher war ich neugierig auf bestimmte Anwendungen, bei denen UX denen dieser Maschine sehr ähnlich ist, aber es fühlt sich irgendwie komisch an. Jetzt weiß ich, dass es zittert.

  • Ich schreibe zum ersten Mal und bekomme zwei Apps (ich habe große Probleme beim Einrichten von React Native mit Android).
  • 0.9 Bei der Hotinstallation ist ein Fehler aufgetreten
  • Die Entwicklung von Visual Studio Code ist ziemlich gut, was mich an die Vergangenheit von Golang erinnert
  • Hässlicher Codestil (Ist das ernst gemeint? Dies ist ein offizielles Beispiel)

Auf jeden Fall war ich wirklich beeindruckt, es übersprang die “nativen Komponenten” und schrieb eine eigene 2D-Rendering-Engine. Viel talentierter als React Native.

Daher denke ich persönlich, dass Flutter mehr Potenzial hat als React Native. Und es gibt viele Unternehmen, die auf lokale Produkte zurückgreifen.
