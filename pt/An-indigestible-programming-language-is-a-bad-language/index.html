Uma linguagem de programação indigesta é uma linguagem ruim

Postado em 2014-12-23 Edited on 2020-12-21 Disqus:
Symbols count in article: 2.5k Reading time ≈ 2 mins.

Quando eu era Gap Year, eu vi este livro. Estava escrevendo perguntas no codecademy. Eu só vi o capítulo Ruby. Naquela época, eu senti que sabia que Go já era muito Geek.

Um ano depois, quando a empresa em que trabalho atualmente estava reestruturando o back-end, pensei em Scala. Um dos meus amigos viu o alto salário do jogo, então ele foi resolutamente verErlang. Ouvi falar de um evento da Thoughtworks. Clojure`

Quando li este livro novamente, descobri que Ruby, Io, Prolog, Scala, Erlang, Clojure, Haskell

Estive sentado no poço e olhando para o céu, li este livro novamente e descobri que realmente não preciso ler este livro

Portanto, se você está curioso e satisfaz sua curiosidade, dê uma olhada. Se quiser estudar seriamente, ignore.

Olhando para a linguagem de programação, existem tantas que você não pode escapar das seguintes categorias:

  1. modelo
  • Funcional
  • OO
  • protótipo
  1. Algoritmo de estrutura de dados
  2. estrutura de dados
    • Tipos de
    • Lista ligada
    • Tabela de hash
  3. processo
    • julgamento
    • ciclo
  4. Concorrente

Em relação à simultaneidade, porque o processo não está de acordo com o pensamento humano, parece não haver uma boa solução no momento
Eu acho que mesmo se for resolvido no futuro, deve ser escrito e encapsulado por algumas pessoas talentosas, para que pessoas com QI medíocre possam resolvê-lo tão simples quanto if outra vez

Pessoalmente, a popularidade de um idioma é inseparável dos seguintes pontos:

  1. Promoção Comercial (Java | OC)
  2. Seguro e estável
  3. É fácil para os humanos lerem

Não há necessidade de dizer os dois primeiros pontos, se você falar muito, vai rasgar

É mencionado nas sete línguas de sete semanas:
De muitas maneiras, Clojure é o sábio mestre de kung fu, o oráculo na colina ou o enigmático treinador Jedi.

Os benefícios do Lisp são conhecidos por “The Hacker and the Painter”, e Emace é conhecido como “o editor de Deus”

No entanto, é tão bom e tão incrível, mas não é popular, é uma questão que vale a pena ponderar

Do ponto de vista do açúcar sintático, ainda me lembro que usei Swift uma vez, um programador de geração anterior (C / C ++ / Java) (não depreciativo), apenas vi o valor de retorno alterado para ->, o tipo de variável era Exclamou por trás
Quanto a mim, aceito completamente Go Swift e outros açúcares gramaticais, mas também acho que as expressões prefixadas deLisp são todas desconfortáveis. A simplicidade é boa, e fico tonto quando é longa, e os colchetes são muito heréticos.
Pode-se ver como é difícil para os programadores aceitar a nova sintaxe depois de se acostumar com um sistema

“O código é escrito para que as pessoas vejam”

Portanto, acho que a linguagem que é a leitura anti-humana irá desaparecer, pelo menos não se tornará popular entre os humanos.

Então Mazi escolheu outra abordagem, focando em como deixar os programadores felizes em escrever código

Portanto, não há melhor linguagem, apenas a linguagem mais adequada

Claro, nada do mencionado acima se aplica a uma situação em que os computadores evoluíram para serem capazes de escrever código

Porque isso rompeu com o paradigma humano
