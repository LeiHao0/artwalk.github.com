Enfin terminé

Pas étonnant que Ruby vous donne la liberté. Du seul point de vue du sucre syntaxique, il est meilleur que Python
Je n’ai pas aimé Ruby à cause du problème «fin», mais je l’ai fait dans son ensemble:

  1. Il n’est pas nécessaire de prêter attention à l’indentation partout comme Python, et elle peut être automatiquement indentée (Python ne l’a jamais vue), alors vous inquiétez
  2. : les symboles etc. sont réduits, les appels de fonction sont plus naturels
  3. Plus orienté objet (Python a toujours le concept de type (int str …), les paramètres de passage, les valeurs de retour, etc. sont particulièrement évidents; Ruby est affaibli, et vous ne pouvez même pas ressentir l’existence de types lorsque vous l’utilisez)

Dans son ensemble, il est réalisé

  1. L’algorithme n’a rien à voir avec le code, bien moins que Python
  2. Il est plus naturel d’écrire

Par conséquent, il est préférable de ne pas utiliser de jugements subjectifs avant de comprendre quelque chose

PS:
Java et Android sont utilisés depuis près d’un an, mais il n’y a pas d’amour, je ne comprends pas cela.
