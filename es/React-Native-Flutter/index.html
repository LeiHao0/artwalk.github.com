作为一个愉快地遵循  步骤的 iOS 开发人员，当我涉足 iOS 开发人员之初，便使用了 Swift，Storyboard 和 AutoLayout。

我认为这就是为什么大多数本机开发人员甚至都不愿意学习 React Native 的原因。

两年前，我建立了 Mattermost 并在以前的公司中进行了一些更改以取代 Slack，这是我接触的第一个 React Native 项目。

不幸的是，丢失代码非常容易，而且每次我想更改某些内容时，我都必须搜索整个项目，没有 Storyboard 让我知道谁告诉谁。

我和我以前的同事聊天，后者是高级前端。他帮助我启动了更新服务器，该服务器也是用 Node.js 编写的。我离开时将这个项目移交给他。

据他介绍，这是因为他不知道如何编写本机代码，就像 React React 类似于 React.js 一样，为什么不使用它同时尝试 iOS 和 Android。然后我问他是否知道如何快速发展，他会喜欢哪种技术。您可以猜出答案 😄。

因为从我的角度来看，有时即使在快速代码中也难以实现功能，这绝对是另一项技术要实现的漫长的路要走。

然后，这几天有机会进行一个演示，其中包含嵌套的磁铁滚动视图，按钮和导航。我不再使用本机代码，而是走出舒适区，尝试使用 React Native 和 Flutter。

  • Views
    • ListView, ScrollView, Labels, Texts, TextFields
  • Actions
    • Drag, Click, Scroll
    • Callback Block or Delegate on view events

因此，如果您精通其中任何一种，剩下的就是 Google 如何在该框架中实现它或阅读其手册。

React Native

React Native 令人印象深刻的是 state

它可以通过 Swift 中的变量的didSet来实现，还有很多框架可以做到，例如 RxSwift，PromiseKit

Flutter

我曾经很好奇某些应用程序 UX 非常像由本机开发的，但以某种方式感到怪异。现在我知道它在颤抖。

  • 这是我第一次编写一次并获得两个应用程序（使用 Android 设置 React Native 时遇到了很多麻烦）
  • 0.9 在热装上有错误
  • 在 Visual Studio Code 上的开发相当不错，这让我回想起 Golang 的过去
  • 丑陋的代码风格（他们对此很认真吗？这是一个官方示例）

无论如何，我真的给它留下了深刻的印象，它跳过了本机组件，并编写了自己的 2D 渲染引擎。比 React Native 天才得多。

因此，就我个人而言，我认为 Flutter 比 React Native 更具潜力。以及有许多公司回滚到本地。
