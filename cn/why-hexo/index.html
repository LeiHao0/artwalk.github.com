Why Hexo

发表于 2013-11-21 更新于 2023-08-07 Disqus：
本文字数： 869 阅读时长 ≈ 1 分钟

第一次搭建的时候用的是 Jekyll，Octopress 没试过

Jekyll 的框架设计分为 代码 + 生成网页 两部分

上传到 github 的时候 push 上去的是代码部分，而 hexo 是本地生成好的网页，速度上应该会有一定的优势

看到默认模板也比较漂亮，所以昨天换成了 hexo

感觉 hexo 最大特点是隐藏了繁琐的各种命令

用 Jekyll 的话，从搭建到完成至少十几条吧

而写新文章至少要

也许在不少人看来，已经很简单了

但要知道有些人并不是攻城狮，他只是想不用脑子搭个博客，重点是在 写作 上面

而写之前要帕拉帕拉打那么多命令，万一脑抽忘了一条，或者少个参数，还得-h，万一英文不好，还得 google，被墙，还得 baidu，等到真正写作的时候，不知道都几个番茄钟了

并且 jekyll 至少要有点 git 的概念，如果写中文博客的话，还要去改 ruby（utf-8）

天天天天天天天哪！

而 hexo 真是简单到爆啊，全程仅仅需要知道 8 个字母

对于我等非专业攻城狮来说，用起来真是爽到泪流满面呐

使用Github页面作为博客

终于，我找到了这里，因为几个月前我就想有一个自己的博客

这里有很多公共博客，比如豆瓣、新浪，但没有一个是我喜欢的

我只是想找一个没有人认识我的地方，可以记录一些我的奇葩经历或者想法

QQ 空间是我长期使用的博客，但里面的人都是我的朋友或同事。我不希望被人当成精神病。^_^

我按照jekyllrb的方法，一步一步的操作，似乎成功了，页面上线了！我的网站上有一个 “Hexo“。

(现在，我用Hexo代替 jekyll)

其实早在一年前，我就从一位同学那里听说github可以作为 Blog 使用。但是这个任务被一天天的拖了下来。我想，一定有很多配置要设置。不但我不喜欢复杂的东西，而且作为一个拖延症患者。

英语快把我逼疯了，我还没有足够的能力去做这件事
