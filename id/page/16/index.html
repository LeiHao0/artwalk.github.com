2 Pengantar singkat tentang NTFS

2.1 NTFS

NTFS (Sistem File Teknologi Baru) æ˜¯ Windows
Sistem file standar NT dan Windows yang lebih baru. NTFS menggantikan sistem file tabel alokasi file (FAT) dan menyediakan sistem file untuk sistem operasi Microsoft seri Windows. NTFS telah melakukan beberapa perbaikan pada FAT dan HPFS (Sistem File Kinerja Tinggi), misalnya, mendukung metadata dan menggunakan struktur data tingkat lanjut untuk meningkatkan kinerja, keandalan, dan pemanfaatan ruang disk. ^ [2] ^

2.2 Situasi saat ini

Dengan Windows berbasis NT
Dengan popularitas 2000 / XP, banyak pengguna individu mulai menggunakan NTFS. NTFS juga menggunakan cluster sebagai unit untuk menyimpan file data, tetapi ukuran cluster di NTFS tidak bergantung pada ukuran disk atau partisi. Menyusut ukuran cluster tidak hanya mengurangi pemborosan ruang disk, tetapi juga mengurangi kemungkinan fragmentasi disk. NTFS mendukung fungsi manajemen enkripsi file, yang dapat memberikan tingkat jaminan keamanan yang lebih tinggi kepada pengguna. ^ [3] ^

Log USN sama dengan direktori WORD dan menyediakan indeks. Tentu saja, jika konten artikel berubah, log USN akan merekam saat modifikasi dilakukan, tetapi tidak mencatat apa yang diubah di dalamnya, jadi file indeksnya sangat kecil. Dan ketika Anda ingin mencari paragraf tertentu dari artikel tersebut, Anda tidak perlu memutar roda mouse secara liar, cukup lihat daftar isi secara langsung, dan Anda hanya perlu menahan ctrl + klik mouse untuk mencari lokasi.

Dengan cara yang sama, ketika Anda ingin mencari file tertentu, Anda dapat langsung mengetahui apakah file tersebut ada dengan mencari di log USN (yaitu, indeks yang ditetapkan).

4.4 Bergabung dengan utas

4.4.1 Mengapa menggabungkan utas

Manfaat utas: utas harganya murah. Thread dimulai lebih cepat, keluar lebih cepat, dan berdampak lebih kecil pada sumber daya sistem.

Pertimbangkan juga:

  1. Karena keterbatasan io, tidak masuk akal menggunakan beberapa utas untuk membaca MFT secara bersamaan di hard disk yang sama.
  2. Akses disk ke MFT yang terletak di partisi berbeda dari hard disk yang sama dapat mempengaruhi kecepatan. Untuk komputer dengan banyak hard disk, efisiensi harus sangat ditingkatkan. Namun, pengguna beberapa hard drive dapat membentuk larik RAID. Solusi terbaik adalah membaca disk A-Z secara berurutan.
  3. Saat MFC pertama kali dibuat, dijalankan secara berurutan, akibatnya antarmuka hanya dapat ditampilkan setelah semua data statistik selesai, Dalam proses ini, pengguna dapat dengan mudah kehilangan kesabaran.
  4. Akhirnya memutuskan thread UI, statistik latar belakang thread pekerja.
