 단계를 기꺼이 따르는 iOS 개발자로서 iOS 개발자를 처음 접했을 때 Swift, Storyboard 및 AutoLayout을 사용했습니다.

이것이 대부분의 네이티브 개발자가 React Native를 배우고 싶지 않은 이유라고 생각합니다.

2 년 전 Mattermost를 설립하고 Slack을 대체하기 위해 이전 회사에서 약간의 변경을가했는데, 이것이 제가 처음 접한 React Native 프로젝트였습니다.

안타깝게도 코드를 잃어 버리기 쉽고 무언가를 변경하고 싶을 때마다 전체 프로젝트를 검색해야합니다. 누가 누가 말했는지 알려주는 스토리 보드가 없습니다.

나는 고급 프론트 엔드였던 이전 동료와 대화를 나눴다. 그는 Node.js로 작성된 업데이트 서버를 시작하도록 도와주었습니다. 내가 떠날 때이 프로젝트를 그에게 넘겼습니다.

그에 따르면 React React가 React.js와 유사한 것처럼 네이티브 코드를 작성하는 방법을 모르기 때문에 iOS와 Android를 모두 사용해 보는 것은 어떨까요? 그런 다음 나는 그가 빨리 개발하는 방법과 그가 어떤 종류의 기술을 원하는지 알고 있는지 물었습니다. 답을 추측 할 수 있습니다 😄.

제 관점에서는 빠른 코드로도 기능을 구현하는 것이 어려울 때가 있습니다. 이는 다른 기술을 달성하기에는 확실히 먼 길입니다.

그런 다음 요즘에는 중첩 된 마그넷 스크롤 뷰, 버튼 및 내비게이션으로 데모를 수행 할 기회가 있습니다. 네이티브 코드를 사용하는 대신 편안한 영역에서 벗어나 React Native와 Flutter를 시도했습니다.

  • 견해
    • ListView, ScrollView, 레이블, 텍스트, TextFields
  • 행위
    • 드래그, 클릭, 스크롤
    • 콜백 차단 또는보기 이벤트 위임

따라서 이들 중 어느 하나에 능숙하다면 Google이 프레임 워크에서이를 구현하거나 설명서를 읽는 방법 만 남습니다.

네이티브 반응

React Native의 인상적인 점은 ** state **입니다.

Swift에서 변수의didSet을 통해 달성 할 수 있으며,이를 수행 할 수있는 프레임 워크가 많이 있습니다 (예 : RxSwift, PromiseKit)

실룩 거리다

UX가이 머신에서 개발 한 것과 매우 유사하다는 특정 애플리케이션에 대해 궁금했던 적이 있지만 어떤면에서는 이상하게 느껴집니다. 이제 나는 그것이 흔들리고 있다는 것을 압니다.

  • 두 개의 앱을 작성하고 얻는 것은 이번이 처음입니다 (Android에서 React Native를 설정하는 데 많은 어려움이 있습니다)
  • 0.9 핫 설치에 오류가 있습니다.
  • Visual Studio Code의 개발은 상당히 훌륭하여 Golang의 과거를 생각 나게합니다.
  • 추악한 코드 스타일 (진지한가요? 이것은 공식적인 예입니다)

어쨌든 나는 그것에 정말 감명을 받았다. 그것은 ‘네이티브 컴포넌트’를 건너 뛰고 자체 2D 렌더링 엔진을 작성했다. React Native보다 훨씬 더 재능이 있습니다.

따라서 개인적으로 Flutter는 React Native보다 더 많은 잠재력을 가지고 있다고 생각합니다. 그리고 많은 회사가 현지로 롤백합니다.
