How to install reaver on Mac OS X

Posted on 2014-05-11 Edited on 2020-12-21 Disqus:
Symbols count in article: 1.6k Reading time ≈ 1 mins.

Update: 2017-05-24 09:03:10

Sorry, this post, I didn’t realize its ** danger before.

In October last year, my network was overdue, and I re-avered the wifi that was used temporarily during the National Day holiday that the new network needed.

When I played the Open Yale Course-Listening to Music on the NAS, I found that my TV (Android system) had a subtitle that said “This website is for my personal use only. Do you have any problems?”.

I was taken aback by this sentence. My NAS has a PandoraBox system. I use it as a bridge. Normally, my TV shouldn’t appear on their router. Obviously, I found that this wifi was a fishing wifi, and my TV was hacked.

So, if some stupid applications are still using http, all my data will be exposed. Even https has a problem with MitM. ‘The product says iOS and macOS have a login trust certificate in the Keychain, but what is Android? Some devices have their own back door.

As for my TV, I think it uses adb connect <TV IP address> which is Android Debug Bridge (adb) Wireless Debugging Over Wi-Fi.

However, I later changed most of the passwords.

So, you can try the reaver of technical exploration, but cannot use it.

Update: 2015-07-27 13:14:32

I wrote it on 2014-05-11, but it was a year late, and now you can easily install reaver with Homebrew.

Thanks Rustam for the suggestion

I always use beini in vmware, and every time I plug in an extra USB wireless adapter to my computer.

When I had a Mac, I just copied beini’s VMX from Windows to the Mac, which meant I still had to use a wireless adapter.

In fact, minidwep or inflator all use the powerful tool Reaver… you know), while Mac is a unix-based system, you can run Reaver directly.

  1. Download reaver-osx.zip
    (Tips: The original reaver-1.4 cannot use make directly on Mac)
  2. Unzip reaver-osx.zip.

Enjoy, don’t do bad things :-)
