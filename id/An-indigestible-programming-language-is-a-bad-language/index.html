Bahasa pemrograman yang tidak dapat dicerna adalah bahasa yang buruk

Diposting di 2014-12-23 Edited on 2020-12-21 Disqus:
Symbols count in article: 2.7k Reading time ≈ 2 mins.

Ketika saya Gap Year, saya melihat buku ini. Saya menulis pertanyaan di codecademy. Saya hanya melihat bab Ruby. Saat itu, saya merasa Go sudah sangat Geek.

Satu tahun kemudian, ketika perusahaan tempat saya bekerja saat ini sedang melakukan refactoring backend, saya mempertimbangkan Scala. Salah satu teman saya melihat gaji yang tinggi untuk game tersebut, jadi dia dengan tegas pergi untuk melihatErlang. Ada acara sharing Thoughtworks yang saya dengar. Clojure`

Ketika saya membaca buku ini lagi, saya menemukan bahwa Ruby, Io, Prolog, Scala, Erlang, Clojure, Haskell

Saya telah duduk di sumur dan melihat ke langit lagi. Saya membaca buku ini lagi dan menemukan bahwa saya tidak benar-benar perlu membacanya.

Nah jika penasaran dan memuaskan rasa penasarannya, silahkan simak yuk, Jika ingin serius belajar lewati saja.

Melihat bahasa pemrogramannya, ada begitu banyak sehingga Anda tidak bisa lepas dari kategori berikut:

  1. model
  • Fungsional
  • OO
  • prototipe
  1. Algoritma struktur data
  2. struktur data
    • Jenis dari
    • Daftar tertaut
    • Tabel hash
  3. proses
    • pertimbangan
    • siklus
  4. Bersamaan

Mengenai konkurensi, karena prosesnya tidak sesuai dengan pemikiran manusia, sepertinya tidak ada solusi yang baik saat ini
Menurut saya, meskipun nanti diselesaikan, harus ditulis dan dikemas oleh beberapa orang berbakat, sehingga orang dengan IQ biasa-biasa saja bisa menyelesaikannya sesederhana menulis ifelse

Secara pribadi, popularitas suatu bahasa tidak terlepas dari poin-poin berikut:

  1. Promosi Komersial (Java | OC)
  2. Aman dan stabil
  3. Apakah mudah bagi manusia untuk membaca

Tidak perlu mengatakan dua poin pertama, jika Anda mengatakan terlalu banyak maka akan menjadi air mata

Disebutkan dalam tujuh bahasa tujuh minggu:
Dalam banyak hal, Clojure adalah ahli kung fu yang bijak, peramal di atas bukit, atau pelatih Jedi yang penuh teka-teki.

Manfaat Lisp dikenal dari “The Hacker and the Painter”, dan Emace dielu-elukan sebagai “editor of God”

Namun, ini sangat bagus dan sangat mengagumkan, tetapi tidak populer, ini adalah pertanyaan yang perlu direnungkan

Dari perspektif gula sintaksis, saya masih ingat bahwa saya menggunakan Swift sekali, programmer generasi sebelumnya (C / C ++ / Java) (non-derogatory), baru saja melihat nilai kembalian berubah menjadi ->, tipe variabelnya adalah Seru dari belakang
Sedangkan untuk diri saya sendiri, saya sepenuhnya menerima Go Swift dan gula tata bahasa lainnya, tetapi saya juga merasa bahwa ekspresi awalanLisp tidak nyaman. Kesederhanaannya bagus, dan saya pusing jika panjangnya, dan tanda kurung tutupnya sangat sesat.
Dapat dilihat betapa sulitnya bagi programmer untuk menerima sintaks baru setelah terbiasa dengan suatu sistem

“Kode ditulis untuk dilihat orang”

Jadi menurut saya bahasa yang anti bacaan manusia akan hilang, atau setidaknya tidak menjadi populer di kalangan manusia

Jadi Mazi mengambil pendekatan lain, dengan fokus pada bagaimana membuat programmer senang menulis kode

Jadi tidak ada bahasa terbaik, hanya bahasa yang paling cocok

Tentu saja, semua hal yang disebutkan di atas tidak berlaku untuk situasi di mana komputer telah berevolusi untuk dapat menulis kode

Karena ini telah melepaskan diri dari paradigma manusia
