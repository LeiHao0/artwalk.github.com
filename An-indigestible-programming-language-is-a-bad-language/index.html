An indigestible programming language is a bad language

Posted on 2014-12-23 Edited on 2020-12-21 Disqus: Symbols count in article: 2.4k Reading time ≈ 2 mins.

When I was Gap Year, I saw this book. I was writing questions at codecademy. I only saw the Ruby chapter. At that time, I felt that Go was already very Geek.

One year later, when the company I am currently working for was refactoring the backend, I considered Scala. One of my friends saw the high salary of the game, so he resolutely went to see Erlang. There was a Thoughtworks event sharing I heard. Clojure`

When I read this book again, I found that Ruby, Io, Prolog, Scala, Erlang, Clojure, Haskell

I’ve been sitting in the well and looking at the sky, I read this book again and found that I don’t really need to read this book

So if you are curious and satisfy your curiosity, you can take a look. If you want to study seriously, skip it.

Looking at the programming languages, there are so many people who can’t escape the following categories:

  1. model
  • Functional
  • OO
  • prototype
  1. Data structure algorithm
  2. data structure
    • Types of
    • Linked list
    • Hash table
  3. process
    • judgment
    • cycle
  4. Concurrent

Regarding concurrency, since the process does not conform to human thinking, there seems to be no good solution at present
I think that even if it is solved in the future, it should be written and encapsulated by some talented people, so that people with mediocre IQ can solve it as simple as writing ifelse

Personally, the popularity of a language is inseparable from the following points:

  1. Commercial Promotion (Java|OC)
  2. Safe and stable
  3. Is it easy for humans to read

No need to say the first two points, if you say too much, it will be tears

It is mentioned in the seven languages of seven weeks:
In many ways, Clojure is the wise kung fu master, the oracle on the hill, or the enigmatic Jedi trainer.

The benefits of Lisp are known from “The Hacker and the Painter”, and Emace is known as “the editor of God”

However, it’s so good and so awesome, but it’s not popular, it’s a question worth pondering

From the perspective of syntactic sugar, I still remember that I used Swift once, a previous generation (C/C++/Java) programmer (non-derogatory), just saw the return value changed to ->, the variable type was Exclaimed from behind
As for myself, I completely accept Go Swift and other grammatical sugars, but I also feel that Lisp‘s prefix expressions are all uncomfortable. The simplicity is good, and I get dizzy when it is long, and the closing brackets are very heretical.
It can be seen how difficult it is for programmers to accept the new syntax after getting used to a system

“Code is written for people to see”

So I think that language that is anti-human reading will disappear, or at least not become popular among humans.

So Mazi took another approach, focusing on how to make programmers write code happily

So there is no best language, only the most suitable language

Of course, none of the above mentioned applies to a situation where computers have evolved to be able to write code

Because this has broken away from the human paradigm
