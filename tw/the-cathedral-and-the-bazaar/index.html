大教堂與集市

Posted on 2014-09-01 Edited on 2020-12-21 Disqus:
Symbols count in article: 3.2k Reading time ≈ 3 mins.

一直以來，我就很奇怪
大眾眼中的程序員，一般是呆頭呆腦，按邏輯辦事，終日沈浸在自己世界，沉默寡言，高智商低情商，為哪種編程語言或框架比另一種要好爭論不休
哦，還有最重要的一條：木有妹紙

這也讓我很困惑，甚至有段時間我發自內心的不認同這個標籤

因為縱觀國外的一些大牛，他們除了代碼精湛之外，還寫 Blog、出書、營銷、開公司、拉風投、潛水、登山、旅行…
而且並非淺嚐輒止，只能用逆天來形容
這…還是程序員麼

看了此書才明白，此人種名曰：黑客(Hacker)

上次看《黑客與畫家》時，就很雞凍
還記得剛學 C 語言時，就和我哥說，不想當一名黑客，也不想搞安全相關的東東
畢竟當時學校那破圖書館裡都是《黑客攻防實踐》《黑客反彙編》之類的東東

時隔多年，現在是知道了，原來我不想成為的是 駭客(cracker)

對於 駭客，Eric 文中給出了一些建議：

如果你想成為一名黑客，請接著讀下去。如果你想成為一名駭客，去讀 alt.2600 新聞組(news:alt.2600)，如果你發現自己並沒有想像中那麼聰明，做好準備去蹲上 5 到 10 次監獄吧。這就是我想對駭客說的。

關於這個，我是存有疑問的，以後有時間看下 2000 年之前他們分別是什麼版本

Java 也是一個不錯的學習編程的語言。它比 Python 要難，但可以產生比 Python 更快的代碼，我想它可以作為一個優秀的第二語言

沒看Python 的直接來看 Ruby 吧，這是一門讓你爽的語言，還有，此書由於出版時間，iOS & Android 尚未出現

最好把這五種語言(Python、Java、C/C++、Perl 和 LISP)都學了。它們不只是最重要的黑客語言，還代表了截然不同的編程方法，每一種都會讓你受益匪淺

重複造輪子

創造性頭腦是無比珍貴的有限資源，它們不應浪費在重新發明輪子這種事上，尤其是還有這麼多迷人的新問題在那裡等著的時候

GUI

問題主要出在人機工程學設計和界面心理學上，黑客在這些方面從來就很弱。黑客在為其他黑客設計接口時做得很好，但他們往往不善於為另外 95%人群的思維過程建模

2. 大教堂與集市

  1. 好的軟件作品，往往源自於開發者的個人需要
  2. 優秀的程序員知道寫什麼，卓越的程序員知道改寫(和重用)什麼
  3. “計劃好扔掉一個吧，遲早你會這麼做的。”(Fred Brooks，《人月神話》第 11 章)
  4. 如果你有正確的態度，有趣的事情自然會找到你。
  5. 當你對一個程序不再感興趣時，你最後的責任就是把它交給一個可以勝任的接棒者
  6. 把你的用戶當成開發合作者對待，如果想讓代碼質量快速提升並有效排錯，這是最省心的途徑。
  7. 早發布，常發布，傾聽用戶的反饋。
  8. 如果有足夠多的 beta 測試者和合作開發者，幾乎所有問題都會很快顯現，然後自然有人會把它解決。
  • Linus 定律道出了大教堂模式和集市模式最關鍵的區別：在大教堂建築者看來，bug 是棘手的、難以發現的、隱藏在深處的，要經過幾個人數月的全心投入和仔細檢查，才能有點信心說已經剔除了所有錯誤。而發布間隔越長，倘若等待已久的發布版本並不完美，人們的失望就越發不可避免。
  • 對集市模式而言則完全不同，在上千名合作開發者熱切鑽研每個新發布版本的情況下，你可以假定 bug 是淺顯易找的，或者至少可以很快變得淺顯易找。所以你會頻繁發布以獲取更多的修正，其副作用是良性的：即便發布中有些小問題，你也不會損失太多。
  1. 聰明的數據結構配上愚笨的代碼，遠比反過來要好得多。
  2. 如果你把 beta 測試者當做最珍貴的資源對待，他們就會成為你最珍貴的資源
  3. 僅次於擁有好主意的是，識別來自用戶的好主意，有時後者會更好
  4. 通常，那些最有突破性和最有創新力的解決方案來自於你認識到你對問題的基本觀念是錯的。
  5. “設計上的完美不是沒有東西可以再加，而是沒有東西可以再減。”
  6. 任何工具都應具備預期內的功能，但一個偉大的工具能給你帶來預期外的功能。
  7. 寫網關類軟件時，盡可能不要干擾數據流，而且絕不要扔掉信息，除非接收方強迫你這麼做
  8. 當你的語言還遠不是圖靈完備(Turing-complete)的時候，語法糖會讓你受益良多。
  9. 系統的安全性只取決於它所擁有的秘密。謹防虛假的秘密。
  10. 想要解決一個有趣的問題，先去找一個讓你感興趣的問題。
  11. 如果開發協調者有一個至少像 Internet 這樣好的溝通媒介，並且知道如何不靠強制來領導，那麼多人合作必然強於單兵作戰
  • 如果你在工作過程中感到恐懼和厭惡(即便你以自嘲的形式來表達——比如懸掛呆伯特玩偶)，就應該意識到過程已經出了問題。快樂、幽默和玩興是真正的資產
  • 玩”是創造性活動中最具經濟效能的工作模式。

3. 開墾心智層

  • 在禮物文化中，社會地位並不取決於你控制了什麼，而是你給予了什麼
  1. 如果它不能像我所預期的那樣工作，那就不是好的——不管它多麼聰明和有原創性。
  2. 在心智層的拓展性工作要比在某功能域內(對現有作品)的重複性工作好。
  3. 能進入主要發行版的作品比不能進入的好。在所有主要發行版中都包含的作品最令人尊敬。
  4. “使用”是最真實的讚美，類別殺手比同類競爭者好。
  5. 相比那些只挑有趣和簡單工作的人，長期致力於艱苦和乏味工作(如調試、寫文檔)的人更令人欽佩。
  6. 重要的功能擴展比低層次的修補好。

4. 魔法鍋

  1. 可靠性/穩定性/可擴展性非常重要。
  2. 除了獨立的同行評審，沒有其他便捷易行的方法驗證設計和實現的正確性。
  3. 該軟件對客戶的業務非常關鍵。
  4. 該軟件創建或運轉一個公共計算或通信基礎架構。
  5. 關鍵方法(或能實現同等功能的方法)屬於公共知識

5. 黑客的反擊

  • 理論和語言的缺乏在兩個方面對我們不利：一是我們不能係統性地思考如何改進我們的方法，二是我們難以向別人解釋和推銷我們的方法

附錄 A：如何成為一名黑客

黑客搞建設，駭客搞破壞

黑客的態度

  1. 這個世界充滿了迷人的問題等待人們去解決。
  2. 不要解決一個問題兩次。
  3. 無聊和乏味是有害的
  4. 自由是好事
  5. 態度不能代替能力。

黑客的基本技能

  1. 學習編程
  • 想要獲得將知識轉化為技能的思維模式，只能通過實踐和跟隨大師。做法就是讀代碼和寫代碼
  1. 找一個開源 UNIX，學習使用和運行它。
  2. 學習如何使用 WWW 和寫 HTML。

地位之於黑客文化

  1. 寫開源軟件。
  2. 協助測試和調試開源軟件
  3. 發布有用的信息。
  4. 幫助做一些基礎工作。
  5. 服務黑客文化自身。

格調問題

  • 學習很好地使用你的母語寫作。儘管有一種陳舊的看法認為程序員寫作不行，但很多黑客(包括我所知道的那些最優秀的黑客們)寫得一手好文章，而且其數量多得讓人吃驚。

  • 閱讀科幻小說，參加科幻小說集會(這是認識黑客和潛在黑客的好方法)。

  • 研習禪修，並且/或者學習武術。 (心智訓練似乎在很多重要方面都是類似的。)

  • 培養起對音樂的鑑賞力。學會欣賞一些獨特類型的音樂。學習很好地演奏樂器，或者學習如何唱歌。

  • 培養出你對雙關語和俏皮話的欣賞能力。

    但這些事與左右腦技能的結合有關，這可能是問題的關鍵所在(黑客不僅需要有很強的邏輯推理能力，而且要能夠很快從問題的具體邏輯中跳脫出來)

不要

  • 不要使用愚蠢、浮誇的用戶名或暱稱。
  • 不要捲入 Usenet(或其他任何地方)的網絡罵戰。
  • 不要自稱為“電腦朋克”，也不要浪費時間交往那些自稱電腦朋克的人。
  • 不要讓你的貼子或 email 充滿錯誤的拼寫和糟糕的語法。

最後：
“只有其他黑客認為你是黑客時，你才是黑客”
所表達出來的價值觀，類似於中世紀歐洲騎士時代精英團體所宣稱的理想(通常總是無法實現)，這些社會精英有足夠的財力超脫於所處的稀缺經濟。像人們期望的那樣，一個有抱負的騎士為正義而戰，他追求榮譽而非獲取錢財，他站在弱勢和受壓迫者一邊，不斷尋求機會挑戰自己英勇才能的極限。也正因如此，他會認為自己(並被別人認為)是人中翹楚——前提是他的能力和品行已獲得其他騎士的承認和正式認可。在亞瑟王傳奇(Arthurian tales)和武功歌(chansons de geste)所頌揚的騎士精神中，我們看到了理想主義、對自我的不斷挑戰和對地位的追求，這與當今激勵黑客的要素是類似的。
