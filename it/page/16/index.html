Individuazione rapida dei file su disco NTFS

Sommario

Questo articolo introduce come enumerare i nomi di tutti i file e le cartelle sul disco rigido nel formato del disco Windows NTFS e utilizzare l’STL di C ++ per costruire una tabella hash, nonché la GUI e i thread di lavoro di MFC, e infine realizzare l’implementazione in base alle parole chiave inserite dall’utente Cerca in modo semplice come la ricerca per parole chiave di Google, quindi restituisci immediatamente tutti i file / cartelle corrispondenti e ottieni ricorsivamente il percorso di sistema del file / cartella.

** Parole chiave *: * NTFS, veloce, parola chiave, ricerca, percorso file **

1. Introduzione

La struttura delle directory di Windows Nei volumi NTFS, i file sono
Disporre sotto forma di albero B +, premere durante la ricerca di file nella directory
Il metodo di ricerca dell’albero B + cerca prima il nodo radice (a partire dalla directory radice), quindi confronta il nome del file da trovare con il nome del file corrispondente al nodo figlio nel nodo radice per determinare quale nodo figlio corrisponde all’area di archiviazione da cercare, quindi Utilizza il nodo figlio come nodo radice corrente e cerca di nuovo finché il file non viene trovato. ^ [1] ^

Sebbene la ricerca fornita dal sistema Microsoft possa cercare contenuti di testo, la velocità è molto insoddisfacente. Nella maggior parte dei casi, vogliamo solo sapere in quale cartella del computer è archiviato il file e questo programma lo risolve molto bene.

2 Breve introduzione a NTFS

2.1 NTFS

NTFS (New Technology File System) 是 Windows
Il file system standard di NT e versioni successive di Windows. NTFS sostituisce il file system della tabella di allocazione dei file (FAT) e fornisce un file system per i sistemi operativi della serie Windows di Microsoft. NTFS ha apportato numerosi miglioramenti a FAT e HPFS (High Performance File System). Ad esempio, supporta i metadati e utilizza strutture di dati avanzate per migliorare le prestazioni, l’affidabilità e l’utilizzo dello spazio su disco. ^ [2] ^

2.2 Situazione attuale

Con Windows basato su NT
Con la popolarità di 2000 / XP, molti utenti individuali hanno iniziato a utilizzare NTFS. NTFS archivia anche i file di dati in cluster, ma la dimensione dei cluster in NTFS non dipende dalla dimensione del disco o della partizione. La riduzione delle dimensioni del cluster non solo riduce lo spreco di spazio su disco, ma riduce anche la possibilità di frammentazione del disco. NTFS supporta la funzione di gestione della crittografia dei file, che può fornire agli utenti un livello più elevato di garanzia di sicurezza. ^ [3] ^

3.1 Idee di design

Legge tutti i nomi di file del disco come dati e abbina i dati in base alle parole chiave inserite dall’utente. Il percorso corretto viene restituito all’utente insieme al percorso per facilitare l’apertura del file.

Il più grande vantaggio della tabella hash è che riduce notevolmente il tempo impiegato per l’archiviazione e la ricerca dei dati, che può essere quasi considerato come un tempo costante; il costo è solo per consumare più memoria. Tuttavia, con sempre più memoria disponibile, vale la pena scambiare lo spazio con il tempo. Inoltre, anche la facilità di codifica è una delle sue caratteristiche.

3.2 Cos’è USN

La massima priorità del programma è leggere l’USN.

USN 是 Aggiorna giornale o modifica numero di servizio
L’abbreviazione di Journal, la funzione di registrazione delle informazioni modificate nel volume NTFS, il numero di file e directory da monitorare possono essere impostati nella partizione e possono essere registrati il ​​tempo di modifica e il contenuto dell’oggetto monitorato.

Quando il registro USN è abilitato, per ogni volume NTFS, quando sono disponibili informazioni sull’aggiunta, l’eliminazione e la modifica di file, NTFS utilizza il registro USN per registrarlo e archiviarlo come
Il formato di USN_RECORD.

3.3 Perché è veloce

Il log USN è equivalente alla directory WORD e fornisce un indice. Ovviamente, quando il contenuto dell’articolo cambia, il log USN registrerà quando viene apportata la modifica, ma non registra ciò che viene modificato in esso, quindi il file index è molto piccolo. E quando vuoi trovare il paragrafo specifico dell’articolo, non è necessario ruotare selvaggiamente la rotellina del mouse, basta guardare direttamente il sommario e devi solo tenere premuto ctrl + clic del mouse per individuare.

Allo stesso modo, quando si desidera trovare un determinato file, è possibile scoprire direttamente se il file esiste effettuando una ricerca nel registro USN (ovvero l’indice stabilito).

PS: sebbene Windows non sia “tutto è un file” (Unix / Linux
Una delle filosofie di base di), ma la cartella esiste anche sotto forma di file, quindi puoi anche trovare la posizione tramite USN.

4.4 Partecipa al thread

4.4.1 Perché unire i thread

I vantaggi dei thread: i thread sono economici. Il thread si avvia più velocemente, esce più velocemente e ha un impatto minore sulle risorse di sistema.

Considera anche:

  1. A causa della limitazione di io, non ha molto senso utilizzare più thread per leggere MFT simultaneamente sullo stesso disco rigido.
  2. L’accesso al disco a MFT che si trova in diverse partizioni dello stesso disco rigido può invece influire sulla velocità. Per i computer con più dischi rigidi, l’efficienza dovrebbe essere notevolmente migliorata. Tuttavia, gli utenti di più dischi rigidi possono formare un array RAID. La soluzione migliore dovrebbe essere quella di leggere i dischi A-Z in sequenza.
  3. Quando l’MFC è stato creato per la prima volta, è stato eseguito in sequenza. Di conseguenza, l’interfaccia può essere visualizzata solo dopo che tutte le statistiche dei dati sono state completate.In questo processo, gli utenti possono facilmente perdere la pazienza.
  4. Infine, decidere un thread dell’interfaccia utente, una statistica di background del thread di lavoro.

4.5.2 Custodia, ordine

Ci sono generalmente le seguenti due situazioni:

  1. L’utente potrebbe non ricordare le maiuscole e le minuscole del nome del file o del percorso
  2. A volte sono presenti troppi file e per filtrare i file corretti potrebbe essere necessario l’uso di lettere maiuscole
  3. Non ricordo l’ordine delle parole chiave

Quindi fornisci le seguenti due opzioni:

  1. Caso rigoroso
  2. Nessun ordine
