带有 Staxrip 的 CUDA GPU 加速了 h264 / h265 / HEVC 视频编码

发表于 2018-10-18 更新于 2023-08-07 Disqus：
本文字数： 520 阅读时长 ≈ 1 分钟

游戏改变了我的生活。

我去过很多年的城市，这些城市曾经有过刺客信条系列的旅行，其中有“睡狗 2”，“ GTA”等等。
当我真正进入现实世界时，它会产生很大的影响。因此，每次回去时，总是想在游戏中体验一次。

但是，随着年龄越来越大 👴，我玩游戏的时间也越来越少 👱。

我在演奏 📹 时正在录制 🎮，这可能是个好主意。

ShadowPlay

由于我有技嘉 Geforce GTX 1080 可以训练深度学习模型，所以我唯一需要的就是 Nvidia ShadowPlay，它非常适合录制游戏。

但是最小比特率是 10Mbps，我想将其降低到 6kbps 以下，因此涉及压缩 🗜️。

Staxrip

我以前总是使用 FFmpeg，但是每次听到 Macbook 的声音时，都会感到慌张，通常花一整夜时间来转换批处理视频。

幸运的是，这次我看到了 Cuda 徽标，这提醒了 GPU 的加速。

然后我发现了staxrip，真是不可思议。

  1. 单击 x264，选择 NVIDIA H.265（我在我的 1080 上测试，它的速度与 NVIDIA H.264 相似，我曾经认为 264 应该比 265 快得多），然后它可以使用 GPU 加速。
  2. 单击编码器选项，将模式更改为 VBR，然后出现“视频比特率”。

CPU 40％，GPU 35％时可以达到 250 帧/秒。
