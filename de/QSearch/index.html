Schneller Speicherort von NTFS-Festplattendateien

Zusammenfassung

In diesem Artikel wird erläutert, wie Sie die Namen aller Dateien und Ordner auf der Festplatte im Windows NTFS-Festplattenformat auflisten und mithilfe der STL von C ++ eine Hash-Tabelle sowie die GUI und die Arbeitsthreads von MFC erstellen und schließlich die Implementierung anhand der vom Benutzer eingegebenen Schlüsselwörter realisieren Suchen Sie so einfach wie die Google-Keyword-Suche und geben Sie dann sofort alle übereinstimmenden Dateien / Ordner zurück und rufen Sie rekursiv den Systempfad der Datei / des Ordners ab.

** Schlüsselwörter *: * NTFS, schnell, Schlüsselwort, Suche, Dateipfad **

1. Einleitung

Die Verzeichnisstruktur von Windows. In NTFS-Volumes sind Dateien
In Form eines B + -Baums anordnen, drücken Sie bei der Suche nach Dateien im Verzeichnis
Die B + -Baumsuchmethode durchsucht zuerst den Stammknoten (ausgehend vom Stammverzeichnis) und vergleicht dann den zu findenden Dateinamen mit dem Dateinamen, der dem untergeordneten Knoten im Stammknoten entspricht, um festzustellen, welcher untergeordnete Knoten dem zu durchsuchenden Speicherbereich entspricht, und dann Verwenden Sie den untergeordneten Knoten als aktuellen Stammknoten und suchen Sie erneut, bis die Datei gefunden wurde. ^ [1] ^

Obwohl die vom Microsoft-System bereitgestellte Suche Textinhalte durchsuchen kann, ist die Geschwindigkeit sehr unbefriedigend. In den meisten Fällen möchten wir nur wissen, in welchem ​​Ordner auf dem Computer die Datei gespeichert ist, und dieses Programm löst dies sehr gut.

2 Kurze Einführung in NTFS

2.1 NTFS

NTFS (New Technology File System) 是 Windows
Das Standarddateisystem von NT und höher Windows. NTFS ersetzt das Dateisystem der File Allocation Table (FAT) und bietet ein Dateisystem für die Betriebssysteme der Windows-Serie von Microsoft. NTFS hat FAT und HPFS (High-Performance File System) mehrfach verbessert. Es unterstützt beispielsweise Metadaten und verwendet erweiterte Datenstrukturen, um die Leistung, Zuverlässigkeit und Speicherplatznutzung zu verbessern. ^ [2] ^

2.2 Aktuelle Situation

Mit dem NT-basierten Windows
Mit der Popularität von 2000 / XP haben viele einzelne Benutzer begonnen, NTFS zu verwenden. NTFS speichert auch Datendateien in Clustern, aber die Größe der Cluster in NTFS hängt nicht von der Festplatten- oder Partitionsgröße ab. Durch Verkleinern der Clustergröße wird nicht nur die Verschwendung von Speicherplatz verringert, sondern auch die Möglichkeit einer Festplattenfragmentierung. NTFS unterstützt die Dateiverschlüsselungsverwaltungsfunktion, mit der Benutzer ein höheres Maß an Sicherheit erhalten können. ^ [3] ^

3.2 Was ist USN?

Die oberste Priorität des Programms ist das Lesen der USN.

USN 是 Service Number Journal oder Change aktualisieren
Mit der Abkürzung für Journal, der Funktion zum Aufzeichnen der geänderten Informationen im NTFS-Volume, können Sie die Anzahl der zu überwachenden Dateien und Verzeichnisse in der Partition festlegen und die Änderungszeit und den Inhalt des überwachten Objekts aufzeichnen.

Wenn das USN-Protokoll aktiviert ist, verwendet NTFS für jedes NTFS-Volume Informationen zum Hinzufügen, Löschen und Ändern von Dateien und verwendet das USN-Protokoll, um es aufzuzeichnen und zu speichern
Das Format von USN_RECORD.

3.3 Warum ist es schnell?

Das USN-Protokoll entspricht dem WORD-Verzeichnis und stellt einen Index bereit. Wenn sich der Inhalt des Artikels ändert, zeichnet das USN-Protokoll natürlich auf, wann die Änderung vorgenommen wird, aber es zeichnet nicht auf, was darin geändert wird, sodass die Indexdatei sehr klein ist. Und wenn Sie den spezifischen Absatz des Artikels finden möchten, müssen Sie das Mausrad nicht wild drehen, sondern nur direkt auf das Inhaltsverzeichnis schauen und nur Strg + Mausklick gedrückt halten, um zu suchen.

Auf die gleiche Weise können Sie beim Durchsuchen einer bestimmten Datei direkt herausfinden, ob die Datei vorhanden ist, indem Sie das USN-Protokoll (dh den festgelegten Index) durchsuchen.

PS: Obwohl Windows nicht “alles ist eine Datei” ist (Unix / Linux
Eine der Grundphilosophien von), aber der Ordner existiert auch in Form von Dateien, so dass Sie den Speicherort auch über USN finden können.

4.4 Thread verbinden

4.4.1 Warum Threads verbinden?

Die Vorteile von Threads: Threads sind billig. Der Thread startet schneller, wird schneller beendet und hat geringere Auswirkungen auf die Systemressourcen.

Beachten Sie auch:

  1. Aufgrund der Einschränkung von io ist es nicht sinnvoll, mehrere Threads zu verwenden, um MFT gleichzeitig auf derselben Festplatte zu lesen.
  2. Der Festplattenzugriff auf MFT in verschiedenen Partitionen derselben Festplatte kann sich stattdessen auf die Geschwindigkeit auswirken. Bei Computern mit mehreren Festplatten sollte die Effizienz erheblich verbessert werden. Benutzer mehrerer Festplatten können jedoch ein RAID-Array bilden. Die beste Lösung sollte darin bestehen, A-Z-Festplatten nacheinander zu lesen.
  3. Als der MFC zum ersten Mal erstellt wurde, wurde er nacheinander ausgeführt. Infolgedessen konnte die Benutzeroberfläche erst angezeigt werden, nachdem alle Datenstatistiken abgeschlossen waren. Während dieses Vorgangs würden Benutzer leicht die Geduld verlieren.
  4. Entscheiden Sie sich schließlich für einen UI-Thread, eine Worker-Thread-Hintergrundstatistik.

6 Probleme aufgetreten

6.1 USN-Datei kann nicht initialisiert werden

Die O-Festplatte auf dem Computer kann die USN-Protokolldatei nicht initialisieren. Beim Debuggen wird Folgendes festgestellt:

DeviceIoControl gibt 0, GetLastError 0x70 zurück

Überprüfen Sie msdn, der Grund ist ERROR_SHARING_PAUSED

Es stellt sich heraus, dass ich vergessen habe, den folgenden Code zu initialisieren und hinzuzufügen. Das Programm kann normal auf die O-Disk zugreifen

6.2 Das Problem von wchar und char unter Unicode

In Bezug auf die Zeichenfolgenverarbeitung wurde zunächst die Standardbibliotheksfunktionszeichenfolge als Verarbeitungsmethode verwendet, jedoch in Unicode
Das Folgende ist ein Doppelbyte-wchar, das bei der Zeichenfolgenkonvertierung in große Schwierigkeiten geraten ist. Schließlich wird CString als Standardzeichenfolge verwendet und \ _T (“”) wird zum Initialisieren der Zeichenfolge verwendet.
